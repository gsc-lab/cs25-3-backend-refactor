# 💇‍♀️ 美容室予約Webシステム（チーム開発）

## 概要
特定の大学内美容室を想定し、  
顧客が **メニュー・デザイナー・日付・時間** を選択して予約できる  
**美容室予約Webシステム**をチームで開発しました。

## 📌 システムの特徴
- 美容室の予約業務を想定した Webアプリケーション
- Client / Designer / Manager の **役割別機能分離**
- 実運用を意識した  
  **予約重複チェック・時間計算ロジック** を実装

---

## 開発背景・目的

### 背景
大学内の美容室では、対面中心の予約方式により、以下の課題がありました。
- 予約待ちが発生しやすく、長時間の待機が必要
- 営業時間や施術内容を事前に確認しづらい
- 事前予約ができず、現地訪問が必須

### 目的
利用者と運営側の双方にとって  
**分かりやすく、使いやすい予約環境**を実現することを目的としました。

- 予約状況の可視化による業務効率向上
- 情報の一元管理による運営負担の軽減
- 事前予約による利用者利便性・サービス品質の向上

---

## 👤 ユーザー区分（Role）

### Client
- メニュー・デザイナー・日時を選択して予約作成
- 自分の予約履歴を確認・キャンセル

### Designer
- 自分に紐づく予約の確認
- 予約ステータスの変更

### Manager
- デザイナーの休日管理
- 店舗情報・施術情報・ニュース等の管理

---

## 🧩 主な機能

### 予約（コア）
- 複数サービス選択（例：カット + カラー）
- 予約重複チェック（同一時間帯・デザイナー）
- デザイナーの休日を考慮した予約制御
- サービス所要時間を考慮した終了時刻の自動計算
- 予約一覧表示（過去 / 未来）
- 役割別（Client / Designer）表示制御

### その他
- ログイン・会員登録（役割付与）
- サービス / デザイナー / サロン / ニュース情報取得

## 🧑‍💻 担当した役割
- チーム開発において **バックエンドを担当**

- 認証・予約機能の設計・実装
- 各種 CRUD 処理の実装

---

## 🧠 工夫したポイント

- 予約とサービスを柔軟に紐づけられるよう、  
  **Reservation / ReservationService を分離したDB設計**を採用
- 複数サービス選択時に、  
  **各サービスの所要時間を合算し、予約終了時刻を自動算出するロジック**を実装
- クライアントの利便性を考慮し、  
  **予約履歴表示時に施術の合計金額を自動計算して出力**
- 実際の美容室運用を想定し、  
  **予約重複やデザイナーの休日など「予約できないケース」を事前に判定する制御**を実装
