# 💇‍♀️ 美容室予約Webシステム（チーム開発）

## 概要
本プロジェクトは、特定の大学内美容室を想定し、  
顧客が **メニュー・デザイナー・日付・時間** を選択して予約できる  
**美容室予約Webシステム**です。

対面予約に依存していた従来の運用を見直し、  
**予約状況の可視化と業務効率の向上**を目的として、チームで開発しました。

---

## 開発背景
大学内美容室では予約管理システムが導入されておらず、  
対面での予約受付が中心となっていました。

その結果、以下のような課題がありました。

- 待ち時間が発生しやすい
- 事前に空き状況や施術内容を確認できない
- 予約状況の管理が属人的になりやすい

これらの課題を解決するため、  
**実際の利用環境を想定した実用的な題材**として本システムを開発しました。

---

## システムの特徴
- 美容室の実運用を想定した Webアプリケーション
- Client / Designer / Manager の **役割別機能分離**
- 予約業務の中核となる  
  **予約重複チェック・時間計算ロジック** を実装
- データ構造と業務ロジックの整合性を重視した設計

---

## 主な機能

### 予約機能（コア）
- 複数サービス選択（例：カット + カラー）
- 同一時間帯・同一デザイナーにおける予約重複チェック
- デザイナーの休日を考慮した予約制御
- サービス所要時間を合算した終了時刻の自動計算
- 予約一覧表示（過去 / 未来）
- 役割別（Client / Designer）表示制御

### その他の機能
- ログイン・会員登録（役割付与）
- サービス / デザイナー / サロン / ニュース情報の取得

---

## ユーザー区分（Role）

### Client
- メニュー・デザイナー・日時を選択して予約作成
- 自身の予約履歴の確認・キャンセル

### Designer
- 自身に紐づく予約の確認
- 予約ステータスの変更

### Manager
- デザイナーの休日管理
- 店舗情報・施術情報・ニュース等の管理

## 技術スタック

| 分類 | 使用技術 | 補足 |
|---|---|---|
| Backend | PHP 8.2 | 予約ロジック・API 実装 |
| Web Server | Nginx | PHP-FPM へのリクエスト中継 |
| Database | MySQL 8.0 | 予約・ユーザー・サービス管理 |
| Infra / Dev | Docker Compose | 開発環境のコンテナ管理 |
| Router | AltoRouter | 軽量ルーティング |
| その他 | Postman / GitHub | API確認 / ソース管理 |

## 担当範囲
本プロジェクトでは、**バックエンド全般を担当**しました。

- RESTful API の設計・実装
- 認証および予約機能の設計・実装
- 各種 CRUD 処理の実装
- 業務ロジック  
  （予約重複チェック・時間計算・休日判定）の実装

---

## 設計・実装上の工夫
- 予約とサービスを柔軟に紐づけるため、  
  **Reservation / ReservationService を分離したデータベース設計**を採用
- 複数サービス選択時に、  
  **各サービスの所要時間を合算し、予約終了時刻を自動算出**
- 利用者の利便性を考慮し、  
  **予約履歴表示時に施術の合計金額を自動計算**
- 実際の美容室運用を想定し、  
  **予約不可となるケースを事前に判定する制御**を実装
